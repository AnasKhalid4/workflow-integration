# Gig Optimization Guide (anas_982) - June 2026

Based on a live review of your gig pages + how Fiverr's ranking works after
the 2024-2025 algorithm changes (Success Score system, conversion-weighted
search). Keep this file; tick items off as you do them.

## Why impressions went to zero

1. The **new-gig boost** (Fiverr rotates new gigs into search for the first
   weeks) expired. After that, position is earned by clicks and orders - with
   0 orders, the algorithm stops showing the gig.
2. The 2024+ **Success Score** system made visibility conversion-driven; even
   top sellers reported 80-90% impression drops.
3. **Editing a gig** sends it for review and causes a temporary dip - your one
   edit explains a dip, not 3 months. Don't panic-edit; consolidate changes.
4. If impressions are LITERALLY zero (not just low): check Gigs tab for
   "under review / denied / paused", check Out of Office is OFF, and search
   your exact gig title in an incognito browser. If it's not indexed at all,
   contact Fiverr support - sometimes it's a flag or glitch.

## Fix list per gig (do all of one gig's changes in ONE edit)

### 1. Bug-fix gig - URGENT pricing trap
- [ ] Basic package currently says **"10 to 15 small bug or layout issue"
      for $15**. Your own FAQ defines a small bug as an under-1-hour fix.
      A buyer can demand 15 fixes for $15; refusing risks a cancellation or
      bad review that tanks your Success Score. Change Basic to
      **"1 small bug or layout issue - $15 - 24h - 1 revision"**.
      Standard: up to 5 bugs ($45-60). Premium: up to 15 / full audit ($120+).
- [ ] Description bullets are merged into one line ("...data-fetching issues -
      Broken forms, state bugs, and logic errors Layout and responsive
      issues..."). Re-format as clean separate bullets.
- [ ] Missing punctuation: "no black box", "I don't just patch the symptom I
      fix the root cause" - add dashes/periods.
- [ ] Tags: keep `fix website bugs`, `nextjs bug fix`; replace `responsive
      design` (too generic/competitive) with `react bug fix`; add `fix react
      error` and `nextjs error` to use all 5 slots.
- [ ] Add FAQ: "Can you fix bugs in apps built with Lovable / v0 / AI tools?"
      -> "Yes - I do this often." (catches a fast-growing buyer group).

### 2. AI / vibe-coding gig - your best ranking shot
This gig sits in Fiverr's NEW "Vibe Coding -> Development & MVP" category,
which Fiverr is promoting in its main navigation. Competition is far lower
than classic web development. Invest here.
- [ ] Tags currently: ai website builder / ai vibe coding / google ai studio /
      lovable ai app. Drop `google ai studio` (nobody searches it). Use all 5:
      `lovable`, `v0`, `mvp development`, `ai website builder`, `ai vibe coding`.
- [ ] Fix typos in description: "SaaSstyle" -> "SaaS-style",
      "productionready" -> "production-ready".
- [ ] Add a **60-second gig video**: screen recording of you prompting
      Lovable/v0, then the deployed app. At zero reviews, video is your
      strongest trust signal and the biggest conversion lever you're missing.
- [ ] Add 2-3 live demo URLs to the portfolio section.
- [ ] Title (optional, include in the same edit): work "MVP" in, e.g.
      "I will build your MVP web app with Lovable, v0 and AI vibe coding".

### 3. Full-stack gig - too broad to rank
React + Next.js + Node + Django in one title = keyword dilution in the most
saturated category on Fiverr. A 0-review seller cannot rank for "full stack
website".
- [ ] Either niche it: "I will build a SaaS MVP dashboard with Next.js and
      Node.js" (pick ONE stack, ONE outcome), or accept it as a portfolio gig
      and put your energy into gigs 1-2.

### 4. Profile
- [ ] Tagline typo: "Nodjs" -> "Node.js". Buyers at 0 reviews check everything.

### 5. NEW gig to create (free new-gig boost)
- [ ] "I will fix bugs in your Lovable or v0 app and deploy it to production"
      in Vibe Coding -> Troubleshooting & Improvements. Thousands of
      non-technical people build with AI tools, get stuck, and search exactly
      this. Low competition + desperate buyers + you get another new-gig boost.

## Behavior levers (these move ranking more than any edit)

- **Fiverr Briefs**: turn on "Get briefs", match keywords to your gigs, and
  send a custom offer to every relevant brief within 30 minutes. This is the
  #1 first-client channel because it skips search ranking completely.
- **Be genuinely online 18:00-02:00 PKT** (US business hours) via the Fiverr
  app. Reply in under 1 hour - your profile already shows "1 hour", protect it.
- **External traffic**: clicks arriving from outside Fiverr are a positive
  signal. Help people in Lovable/v0 Discords, r/lovable, X build-in-public,
  LinkedIn - answer first, link your gig second. Never spam.
- **Edit discipline**: one consolidated edit per gig, then hands off for 2-3
  weeks while you measure with the tracker.
- **First-client pricing**: $15-30 entry, 24h-2day delivery, over-deliver,
  then politely note that honest feedback helps you (never pressure for
  5 stars - that violates ToS).

## 7-day sprint to the first client

- Day 1: fix the $15/15-bugs trap + typos + tags (one edit per gig).
- Day 2: record the 60s videos, upload demo links.
- Day 3: create the Lovable/v0 bug-fix gig.
- Day 4-7: online every US evening, every Brief answered < 30 min, one genuine
  community share per day, deploy this repo and watch which keywords you
  actually appear for.
